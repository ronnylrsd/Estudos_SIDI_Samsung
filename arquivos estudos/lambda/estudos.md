# Regiões
Conjuntos de Data Centers (AZ) onde é possível utilizar serviços da AWS, projetada para funcionar independente das outras, existe diferença de valores cobrados para o mesmo serviço entre regiões.
US-EAST-1 (região) -> onde os serviços chegam primeiro e é mais barato, onde tem mais pessoas usando.
No console da AWS -> você vê somente os recursos que estão vinculados à região específicada
# Zonas de Disponibilidade (AZ)
- é  um conjunto de data centers, projetado para trabalhar de forma isolada das demais AZ's, sendo redundante e seguro, pelo menos 2 AZ's
- União das AZs -> Região
- Geograficamente distantes para que não ocorra alguma catastofre e todos fiquem indisponíveis
# Zonas Locais
Permitem que você utilize serviços seletos, como os de computação e armazenamento, mais perto dos usuários.
Acesso de baixa latência, conectadas a região principal por meio de uma rede privada de banda larga muito alta e redudente da Amazon.
